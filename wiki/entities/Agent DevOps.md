# Agent DevOps

> 将 Agent 的开发、评测、交付、运营和反馈连接为持续改进闭环。

## 简介

图片用“开发 → 测试 → 构建 → 部署 → 运维 → 迭代”表示 Agent 系统的生命周期。相比普通软件，Agent 的行为具有概率性，因此测试不仅要覆盖代码，还要覆盖任务表现、模型变化、工具交互和安全边界。

## 关键信息

- **类型**：概念 / 工程实践
- **领域**：AI Agent 交付与运营
- **生命周期**：开发、测试、构建、部署、运维、迭代
- **反馈来源**：[[Agent 可观测性]]、用户反馈、离线评测与安全事件
- **相关概念**：[[Agent Runtime]]、[[Agent 安全治理]]

## 工程要点

- 对提示、模型、工具定义、知识库和策略共同做版本管理。
- 建立离线评测、集成测试、安全测试和线上渐进发布机制。
- 把质量、时延、成本、安全和业务价值作为联合发布门槛。
- 保存失败案例并回灌评测集，避免同类回归反复出现。
