# Agent 安全治理

> 对 Agent 的身份、数据、网络、模型行为和操作记录实施端到端约束。

## 简介

Agent 既消费信息，也可能调用工具改变外部世界，因此安全边界必须覆盖输入、推理、执行和输出。图中包含身份认证、RBAC/ABAC、加密与脱敏、网络防护、提示注入防护、输出过滤及审计合规。

## 关键信息

- **类型**：概念 / 横切能力
- **领域**：AI 安全与合规
- **控制面**：身份权限、数据保护、网络隔离、模型防护、审计
- **核心原则**：最小权限、纵深防御、全程可追溯
- **相关概念**：[[Agent Harness]]、[[Agent Runtime]]、[[Agent 可观测性]]

## 工程要点

- 工具权限应按任务、用户和数据域动态收敛，而不是给 Agent 永久高权限。
- 把外部内容视为不可信输入，隔离数据与指令，防范提示注入。
- 高风险动作增加参数校验、策略审批或人工确认。
- 审计记录需要覆盖是谁、何时、依据什么上下文执行了什么动作。
