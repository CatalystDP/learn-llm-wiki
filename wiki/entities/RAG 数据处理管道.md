# RAG 数据处理管道

> 把分散、脏乱、结构各异的原始数据，转化为可检索、可更新、可治理的知识资产。

## 简介

图中把数据层展开为数据源、数据加载、文本分块、向量化、索引构建和知识图谱。这个拆分很关键，因为很多 RAG 项目的问题不是“检不到”，而是上游数据没被正确切分、标注、压缩或增量更新。

## 关键信息

- **类型**：概念 / 数据基础设施
- **领域**：RAG 数据工程
- **关键步骤**：采集、清洗、分块、向量化、索引、更新
- **质量控制**：数据校验、去重、覆盖度分析、时效性管理
- **相关概念**：[[RAG 检索增强]]、[[RAG 评测与优化]]、[[AI Agent 全栈工程]]

## 工程要点

- 分块策略应该服从文档结构和查询任务，而不是只按固定 token 长度机械切割。
- 元数据提取和增量索引决定后续过滤检索、权限隔离和更新效率。
- 向量化不是一次性预处理任务，模型升级或语料变化都可能触发重建成本。
- 知识图谱适合高价值、高关系密度领域，但不应被默认当成所有 RAG 的必要层。
