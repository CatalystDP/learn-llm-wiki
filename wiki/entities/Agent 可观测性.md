# Agent 可观测性

> 让团队能够解释 Agent 做了什么、为什么这样做、效果与代价如何。

## 简介

Agent 可观测性把传统日志、指标和链路追踪扩展到模型推理、上下文、工具调用、用户反馈与业务结果。图片还列出性能分析、告警通知和反馈收集。

## 关键信息

- **类型**：概念 / 横切能力
- **领域**：AI Agent 运维与评估
- **技术信号**：日志、指标、追踪、性能、告警
- **结果信号**：任务成功率、答案质量、用户反馈、成本与时延
- **相关概念**：[[Agent Runtime]]、[[Agent DevOps]]、[[Agent 安全治理]]

## 工程要点

- 同时观察系统是否运行正常和任务是否真正完成。
- 以一次任务为追踪单元，关联模型、检索、工具、状态变更和最终输出。
- 对敏感上下文做脱敏后再记录，避免可观测性系统变成数据泄漏面。
- 将线上失败样本和用户反馈转成离线评测集，驱动迭代。
